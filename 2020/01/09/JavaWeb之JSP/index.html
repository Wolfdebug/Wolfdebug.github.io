JavaWeb之Jsp

九大内置对象

pageContext, request, session, application
out, response, config, page, exception

  1. request: HttpServletRequest 的一个对象.

常用方法：

方法 说明
String getParameter(String name) 根据表单组件名称获取提交数据
String[ ] getParameterValues(String name) 获取表单组件对应多个值时的请求数据
void setCharacterEncoding(String charset) 指定每个请求的编码
RequestDispatcher getRequestDispatcher(String path) 返回一个

  1. response: HttpServletResponse 的一个对象(在 JSP 页面中几乎不会调用 response 的任何方法.)

  2. pageContext: 页面的上下文（当前jsp页面）, 是 PageContext 的一个对象.

    可以从该对象中获取到其他 8 个隐含对象. 也可以从中获取到当前页面的其他信息.

  3. session: 代表浏览器和服务器的一次会话跟踪（session失效，就要重新登录）， 是 HttpSession 的一个对象.

  4. application: 代表当前 WEB 应用. 是 ServletContext 对象.可以获取初始化参数

  5. config: 当前 JSP 对应的 Servlet 的 ServletConfig 对象(开发的时候几乎不使用).

  6. out: JspWriter 对象. 调用 out.println() 可以直接把字符串打印到浏览器上. 这个是最常用的用法，要注意。

  7. page: 指向当前 JSP 对应的 Servlet 对象的引用, 但为 Object 类型, 只能调用 Object 类的方法(几乎不使用)

    out.println(this) 和 out.println(page);输出的对象是同一个。

  8. exception: 只有在特殊的时候用，在jsp页面之间点不出来后面的内容的。在声明了 page 指令的isErrorPage="true" 时, 才可以使用. *
    定义一个error.jsp，<%@ page isErrorPage="true" %>然后在页面写代码<% exception.(对应的处理)%>

四大作用域

pageContext, request, session, application(对属性的作用域的范围从小到大)
在当前作用域找不到某个属性值时，会自动向更大的作用域里面去寻找。

解决中文乱码

中文乱码出现的场景及解决方法：

  1. 在 JSP 页面上输入中文, 请求页面后出现乱码:
  • 保证 contentType=”text/html; charset=UTF-8”, pageEncoding=”UTF-8” charset 和 pageEncoding 的编码一致, 且都支持中文. 通常建议取值为UTF-8
  • 还需保证浏览器的显示的字符编码也和请求的 JSP 页面的编码一致.
  1. 获取中文参数值: 默认参数在传输过程中使用的编码为 ISO-8859-1

  • 对于 POST 请求: 只要在获取请求信息之前(在调用 request.getParameter), 调用 request.setCharacterEncoding(“UTF-8”) 即可.

  • 对于 GET 请求: 前面的方式对于 GET 无效.

    • 治标： 使用参数强转修改

    • 治本：URIEncoding参数指定对所有GET方式请求进行统一的重新编码（解码）的编码。

    • useBodyEncodingForURI参数表示是否用request.setCharacterEncoding
      参数对URL提交的数据和表单中GET方式提交的数据进行重新编码，在默认情况下，该参数为false。

转发与重定向的区别

请求的转发 请求的重定向
地址栏是初次发出请求的地址. 地址栏不再是初次发出的请求地址. 地址栏为最后响应的那个地址
在最终的 request 对象和中转的那个 request 是同一个对象 在最终的 Servlet 中, request 对象和中转的那个 request 不是同一个对象
只能转发给当前 WEB 应用的的资源 可以重定向到任何资源
/ 代表的是当前 WEB 应用的根目录 / 代表的是当前 WEB 站点的根目录. (站点：服务+端口)

辨别“/”代表的路径:

当/由浏览器解析时：表示当前 WEB 站点的根目录. (站点：服务+端口)

当/由后台java代码解析时：表示当前 WEB 应用的根目录. (项目名)

会话机制

session

一类用来在客户端与服务器端之间保持状态的解决方案;

有时候Session也用来指这种解决方案的存储结构。

session的流程
  1. 当程序需要为某个客户端的请求创建一个session时，服务器首先检查这个客户端的请求里是否包含了一个session标识(即sessionId),如果已经包含一个sessionId则说明以前已经为此客户创建过session，服务器就按照session id把这个session检索出来使用(如果检索不到，可能会新建一个，这种情况可能出现在服务端已经删除了该用户对应的session对象，但用户人为地在请求的URL后面附加上一个JSESSION的参数)。
  2. 如果客户请求不包含sessionId，则为此客户创建一个session并且生成一个与此session相关联的sessionId，这个session id将在本次响应中返回给客户端保存。
session的保存机制
  1. session通过SessionID来区分不同的客户, session是以cookie或URL重写为基础的，默认使用cookie来实现，系统会创造一个名为JSESSIONID的输出cookie，这称之为session cookie
  2. 以区别persistent cookies(也就是我们通常所说的cookie),session cookie是存储于浏览器内存中的，并不是写到硬盘上的，通常看不到JSESSIONID，但是当把浏览器的cookie禁止后，web服务器会采用URL重写的方式传递Sessionid，这时地址栏看到
  3. session cookie针对某一次会话而言，会话结束session cookie也就随着消失了，而persistent cookie只是存在于客户端硬盘上的一段文本。
  4. 关闭浏览器（退出浏览器进程），只会是浏览器端内存里的session cookie消失，但不会使保存在服务器端的session对象消失，同样也不会使已经保存到硬盘上的持久化cookie消失。
清除session和session过期

session是保存在服务器端的对象，是由服务器创建的;

关闭（退出）浏览器只会使存储在客户端浏览器内存中的session cookie失效，不会使服务器端的session对象失效。

  1. 直接调用 HttpSession 的 invalidate()
    调用了之后，session对象会被删除，每次访问这个页面，都是一个新的session对象

  2. HttpSession 超过过期时间.

最大时效： 可以在 web.xml 文件中配置 Session 的最大时效, 单位是分钟

返回最大时效: getMaxInactiveInterval() 单位是秒
设置最大时效: setMaxInactiveInterval(int interval)
